# descript

보고서형 입력 문서에서 개발에 필요한 기본 파일과 폴더 뼈대를 추론하는 책임 영역입니다.

## 책임

- 명시적 디렉토리 트리가 없는 보고서를 분석합니다.
- 제목에 기능 의미가 있는 섹션의 하위 제목을 기능 모듈로 해석합니다.
- 제목에 산출물 의미가 있는 섹션의 목록을 산출물 문서 후보로 해석합니다.
- 사용 기술 문장에서 개발 영역을 추론합니다.
- 보고서 첫 제목은 하위 폴더로 만들지 않고 결과 루트 경계명으로 사용합니다.
- 추론 결과를 기존 폴더 생성 엔진이 이해하는 구조 항목으로 반환합니다.

## 생성하는 대표 구조

```text
docs/
modules/<기능명>/
backend/
frontend/
ai/
database/
infra/
tests/
operations/
```

## 주의

이 폴더의 로직은 보고서 내용을 바탕으로 개발 뼈대를 추론합니다. 보고서 제목은 결과 루트 이름으로만 쓰고, 목차 제목은 폴더로 만들지 않습니다. 폴더 구조에 필요한 기능/기술/산출물 의미만 반영합니다. 실제 프레임워크별 완전한 프로젝트 코드를 생성하는 단계는 아니며, 개발 시작 전에 필요한 책임별 자리와 문서 파일을 만드는 단계입니다.
